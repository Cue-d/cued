# Cued

A cloud-based personal CRM with multi-platform messaging (iMessage, Slack, LinkedIn) and AI-powered action suggestions.

## Prerequisites

| Requirement | Version | Check |
|-------------|---------|-------|
| Node.js | 22+ | `node --version` |
| pnpm | 9+ | `pnpm --version` |

**For Electron app (macOS only):**
- macOS 15+, Swift 6.0+ (`swift --version`)
- Full Disk Access and Contacts permissions in System Settings

## Quick Start

```bash
# 1. Install dependencies
pnpm install

# 2. Set up Convex backend
cd packages/convex
npx convex login          # One-time browser auth
npx convex dev            # Creates .env.local with deployment URL
# Note the deployment URL (e.g., dev:your-project-123)

# 3. Set up web app
cd ../../apps/web
cp .env.local.example .env.local
# Edit .env.local - see Environment Variables below

# 4. Start everything
cd ../..
pnpm dev
```

Web app opens at http://localhost:3000


### Run the App

```bash
# After build completes, start the dev server
pnpm dev

# The dev client will connect automatically
# Or scan the QR code with your device
```

### Local Mobile Build

If you prefer building locally instead of EAS cloud:

```bash
cd apps/mobile

# Generate native projects
npx expo prebuild

# Run on iOS Simulator
npx expo run:ios

# Run on physical device (requires code signing)
npx expo run:ios --device
```

### Code Signing for Physical Devices

For physical device testing, you need an Apple Developer account:

1. Open `apps/mobile/ios/Cued.xcworkspace` in Xcode
2. Select the Cued target → Signing & Capabilities
3. Select your Team and let Xcode manage signing
4. Or use EAS with `eas build --profile preview` (handles signing automatically)

## Environment Variables

Each package has its own `.env.local` file (gitignored).

### packages/convex/.env.local

Auto-generated by `npx convex dev`:

```bash
CONVEX_DEPLOYMENT=dev:your-project-name
```

### apps/web/.env.local

```bash
# Required: Convex (convert deployment to URL)
# dev:your-project-name → https://your-project-name.convex.cloud
NEXT_PUBLIC_CONVEX_URL=https://your-project-name.convex.cloud

# Required: WorkOS Auth
WORKOS_CLIENT_ID=client_xxx
WORKOS_API_KEY=sk_xxx
WORKOS_COOKIE_PASSWORD=xxx  # Min 32 chars (generate: openssl rand -base64 32)
NEXT_PUBLIC_WORKOS_REDIRECT_URI=http://localhost:3000/callback

# Optional: Integrations
AI_GATEWAY_API_KEY=xxx        # AI features (Vercel AI Gateway, optional on Vercel deployments)
```

### apps/electron/.env.local (if needed)

```bash
# Uses same Convex URL as web
CONVEX_URL=https://your-project-name.convex.cloud
```

## Troubleshooting

| Problem | Solution |
|---------|----------|
| "Error accessing messages database" | Grant Full Disk Access to your terminal |
| "Contacts access denied" | Grant Contacts access in System Settings |
| Convex types missing | Run `cd packages/convex && pnpm dev` |

### better-sqlite3 Issues (Electron)

Native modules like `better-sqlite3` need to be rebuilt for Electron's Node version.

**"Module not found" or ABI mismatch errors:**

```bash
cd apps/electron
npx @electron/rebuild -w better-sqlite3 --force
```

**Code signing errors on macOS 15+ (Sequoia):**

macOS 15+ requires all native modules to be signed. For local development:

```bash
cd apps/electron
./scripts/sign-dev.sh
```

This uses ad-hoc signing (no certificate required). For production builds, `electron-builder` handles signing automatically.


### Mobile Build Issues

| Problem | Solution |
|---------|----------|
| Code signing errors | Open `.xcworkspace` in Xcode, set Team in Signing & Capabilities |
| Metro bundler issues | `cd apps/mobile && npx expo start --clear` |
