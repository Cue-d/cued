# LinkedIn Messaging Integration - Progress Log

## Overview
Port Beeper's mautrix-linkedin Go client to TypeScript for the Electron app.
Reference: https://github.com/mautrix/linkedin/tree/main/pkg/linkedingo

## Parallel Groups
- P1: Tasks 1.2, 1.3, 1.4 (urn, request, client)

- P4: Tasks 5.1, 5.2 (unit tests, integration tests)

### 2026-01-19 - Task 3.2: Create LinkedInSyncManager
**Files changed:**
- `apps/electron/src/main/sync/linkedin-sync.ts` (new)

**Summary:**
Created LinkedInSyncManager class following sync-manager.ts pattern:
- `_client: LinkedInClient` - LinkedIn API client property with getter/setter
- `convexClient: ConvexHttpClient` - Convex client for data sync
- `conversationCursors: Map<string, string>` - per-conversation pagination cursors
- `conversationSyncToken: string | null` - incremental conversation sync token
- `start()` - begins 5-minute polling interval, runs initial sync immediately
- `stop()` - clears interval
- `runSync()` - main sync cycle: refreshes auth, syncs conversations then messages
- `syncConversations()` - fetches conversations using sync token, processes each one
- `syncMessages(conversationId)` - fetches messages with cursor management
- `sendMessage(conversationId, text)` - sends message via API and syncs to Convex
- `syncConversationToConvex()` / `syncMessagesToConvex()` - transform and sync to Convex (mutations pending task 4.2)
- `getProgress()` - returns current sync progress
- `reset()` - clears all cursors and sync state
- Progress callbacks: `setTokenProvider()`, `setProgressCallback()`, `setAuthInvalidCallback()`
- Auth error detection with `isAuthError()`
- Singleton pattern with `getLinkedInSyncManager()`

Verification: `pnpm typecheck` passes in apps/electron
