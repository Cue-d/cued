### 2026-01-19 - Task 1.4: Port client.ts with LinkedInClient class
**Files changed:**
- `apps/electron/src/main/linkedin-api/client.ts` (new)
- `apps/electron/src/main/linkedin-api/index.ts` (updated export)

**Summary:**
Created LinkedInClient class ported from mautrix-linkedin Go client:
- Private cookies array with setCookies()/getCookie()/getCookieValue() methods
- Private userEntityURN property with getter/setter for authenticated user
- userAgent and xLiTrack properties for request headers
- eventHandlers property for real-time event callbacks
- isAuthenticated() checks for li_at and JSESSIONID cookies
- getSessionId() extracts JSESSIONID value (strips quotes for CSRF token)
- Method signatures defined: getConversations, getConversationsBefore, getMessages, getMessagesBefore, sendMessage, getConnections, searchPeople
- Result types exported: ConversationsResult, MessagesResult, ConnectionsResult, LinkedInClientOptions

Verification: `pnpm typecheck` passes in apps/electron

### 2026-01-19 - Task 1.3: Port request.ts with CSRF and retry logic
**Files changed:**
- `apps/electron/src/main/linkedin-api/request.ts` (new)
- `apps/electron/src/main/linkedin-api/index.ts` (updated export)

**Summary:**
Created AuthedRequest builder class ported from mautrix-linkedin Go client:
- Error classes: LinkedInAuthError (401/403), LinkedInRequestError (other failures)
- getCSRFToken(cookies): extracts token from JSESSIONID, removes surrounding quotes
- formatCookieHeader(cookies): formats cookies for request header
- AuthedRequest class with method chaining:
  - withMethod(), withHeader(), withQueryParam(), withRawQuery()
  - withJSONPayload() for POST bodies
  - withGraphQLQuery() for LinkedIn GraphQL API (query ID + variables)
  - withXLIHeaders() for x-li-track header
  - doRaw() with retry logic: 5 retries, exponential backoff (1s, 2s, 4s, 8s, 16s)
  - doJSON<T>() for typed JSON responses
- Factory functions: newRequest(), newGetRequest(), newPostRequest()
- GraphQL helpers: newMessagingGraphQLRequest(), newVoyagerGraphQLRequest()
- Auth errors (401/403) don't retry; 502/503/504 are retriable

Verification: `pnpm typecheck` passes in apps/electron
